# BepInEx Mod 进度摘要

更新日期：2026-06-11

## 已完成

- BepInEx Unity IL2CPP 插件入口、配置和本地 API。
- 运行时反射读取当前游戏进度、库存、流行标签、经营场景、稀客和稀客订单。
- Tauri 独立伴随窗口，默认由 Mod 自动启动。
- 伴随窗口页面：概览、普客、稀客、经营中、任务、修改、日志。
- 夜间经营当前稀客点单推荐、专注模式、收藏置顶、库存数量显示。
- 经营中可按已摆放厨具过滤当前场景无法制作的料理；严格点单推荐不足时显示明确标注的喜好备选。
- 可选实验性游戏界面置顶，把当前第一笔订单的推荐料理、材料和酒水排到游戏选择列表前方。
- 可选实验性目标厨具高亮，把当前第一笔订单推荐料理所需厨具以黄色脉冲标记出来。
- 实验性自动化：可按当前排序第一笔稀客订单自动取酒、开始料理、收取料理和完成订单；自动开始料理会固定尝试完成原生 QTE 奖励结算。
- 手柄导航：支持页签切换、页面内移动、滑杆调节、收藏操作和焦点切换防抖。
- 任务页：读取当前进度中可接或正在推进的交互任务，包含全局 NPC、当前场景 NPC、`DaySceneMap`、跟踪交互物件、场景交互组件和 `trackingMissions` fallback 来源诊断；支持状态筛选，并尽量显示未接取 NPC 所在场景。
- 修改页：支持当前运行时材料/酒水库存快捷修改。
- 普客订单诊断与自动化：经营中页通过 `普客` 页签展示当前普客订单、桌位、料理、酒水和完成状态，并在开启实验性自动化总开关、启用普客处理和对应普客阶段开关后，自动按首次出现顺序处理最多 3 笔未满足普客订单。
- 游戏关闭或本地 API 断开后，伴随窗口自动退出。
- 一键构建脚本和安装包打包脚本。

## 当前边界

- 不再保留独立网站功能；`pnpm build` 只用于生成伴随窗口前端产物。
- 不再读取 `.memory` 存档文件；运行时状态来自游戏内对象。
- 日志捕获可辅助识别稀客订单，但核心目标是优先依赖运行时对象。
- 自动化为实验性功能，稀客只处理当前排序第一笔未满足订单；普客按首次出现顺序最多并发处理 3 笔未满足订单；全自动整晚经营不在当前范围内。
- 普客订单已接入自动轮询处理，开始料理、收取和出错暂停阶段与稀客订单独立配置；自动开始料理固定尝试完成原生 QTE 奖励结算。普客料理完成后会绑定原目标订单并通过 `IzakayaConfigure.StoreFood()` 收至游戏料理暂存容器，不占用玩家送餐盘，也不自动送达订单。自动化会显示当前步骤、重试和回退状态，目标料理/酒水长时间未进入容器时可回退重试；诊断写入 `BepInEx/config/MystiaStewardCompanion/automation-jobs.log`。

## 后续重点

- 继续补强不同游戏版本下的运行时字段兼容。
- 减少对日志文本的兜底依赖。
- 继续优化伴随窗口的性能、自动化错误恢复和夜间经营刷新节奏。
