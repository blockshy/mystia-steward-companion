# mystia-steward-companion Mod 使用说明

本文档面向普通用户，说明 BepInEx 安装、Mod 安装/更新、伴随窗口使用、常用设置和故障排查。开发、构建和本地 API 说明见 [README.dev.md](README.dev.md)。

Mod 会在读取存档后刷新游戏当前运行时数据。游戏仍在标题、主菜单或加载页面时，伴随窗口可能显示“运行时数据不可用”；进入可操作场景后会自动刷新。

## 安装 BepInEx

1. 找到游戏根目录。Steam 可通过“管理 -> 浏览本地文件”打开，目录中应能看到游戏 `.exe` 和 `GameAssembly.dll`。
2. 打开 BepInEx Bleeding Edge 下载页：<https://builds.bepinex.dev/projects/bepinex_be>。
3. 下载与系统和游戏架构匹配的 Unity IL2CPP 包。当前项目在 Windows 64 位下建议使用已经验证的 #783 构建：

```text
BepInEx-Unity.IL2CPP-win-x64-6.0.0-be.783+c58c42d.zip
```

   BepInEx Bleeding Edge 是开发构建，不同构建之间可能有兼容性差异。本项目当前不建议使用 #784 或之后的构建；如果后续确认新版可用，再按新版说明更新。

4. 将压缩包内容解压到游戏根目录。解压后应出现：

```text
BepInEx/
doorstop_config.ini
winhttp.dll
```

5. 启动游戏一次，等待 BepInEx 生成目录和 IL2CPP interop 程序集。首次启动可能较慢。
6. 关闭游戏，确认以下目录存在：

```text
BepInEx/config/
BepInEx/core/
BepInEx/interop/
BepInEx/plugins/
```

如果没有生成这些目录，通常是下载了非 IL2CPP 包、解压位置错误，或游戏没有成功启动过一次。

## 安装 Mod

发布包文件名通常为：

```text
mystia-steward-companion-bepinex.zip
```

将压缩包中的 `mystia-steward-companion/` 整个目录放入游戏目录：

```text
游戏根目录/
  BepInEx/
    plugins/
      mystia-steward-companion/
        MystiaStewardCompanion.BepInEx.dll
        companion/
          mystia-steward-companion.exe
```

安装后启动游戏。`BepInEx/LogOutput.log` 中应出现类似日志：

```text
mystia-steward-companion 1.x.x loaded. Press F8 to open or focus the companion window.
Local API loopback listener is available at http://127.0.0.1:32145
```

如果只想在另一台设备上运行伴随窗口，不需要把完整 Mod 发布包安装到 B 设备。Windows 设备可下载 Release 中的独立伴随窗口 EXE，下载后直接双击运行，不需要解压：

```text
mystia-steward-companion-companion-windows-x64.exe
```

Android 设备可下载 Release 中对应 CPU 架构的 APK。普通手机和平板优先选择 `arm64-v8a`，只有较旧设备再选择 `armeabi-v7a`：

```text
mystia-steward-companion-android-arm64-v8a.apk
mystia-steward-companion-android-armeabi-v7a.apk
```

这些独立伴随窗口只用于 B 设备连接 A 设备上的游戏；A 设备仍必须按上面的方式安装 BepInEx 和 Mod。

## 更新 Mod

支持自动更新的版本会在设置页显示更新状态。发现新版本后，可以在伴随窗口中点击检查、下载和打开安装程序；更新包会先保存到配置目录，实际替换由独立更新程序在弹窗中提示关闭游戏并显示进度。首次安装或从不支持自动更新的旧版本升级时，仍按下面的手动流程处理。

默认只检查正式版本。预览版仅用于自动更新测试，普通用户不需要开启；测试者如需参与预览版更新链路，可以在配置文件中设置 `Updates.IncludePrerelease = true`。

手动更新时只需要替换插件目录，通常不需要删除配置和收藏数据。

1. 关闭游戏。
2. 确认伴随窗口也已退出；如果仍在托盘中，右键托盘图标并选择退出。
3. 可选备份以下用户数据：

```text
BepInEx/config/com.tyukki.mystia-steward-companion.cfg
BepInEx/config/MystiaStewardCompanion/
```

4. 删除旧插件目录：

```text
BepInEx/plugins/mystia-steward-companion/
```

5. 解压新版 `mystia-steward-companion-bepinex.zip`，把新的 `mystia-steward-companion/` 放回：

```text
BepInEx/plugins/
```

6. 启动游戏，在伴随窗口顶部或 `BepInEx/LogOutput.log` 中确认版本号已变为新版。

注意事项：

- 不要把新包解压成 `BepInEx/plugins/mystia-steward-companion/mystia-steward-companion/` 这种嵌套目录。
- 收藏配方保存在 `BepInEx/config/MystiaStewardCompanion/favorites.json`，自定义推荐料理保存在 `BepInEx/config/MystiaStewardCompanion/custom-recipes.json`，只替换插件目录不会覆盖它们。
- 自动更新使用 GitHub Release 中的 `update-manifest.json` 校验更新包 SHA256；该清单不包含本机打包路径。
- 预览版 Release 不会被默认更新检查命中；只有手动开启 `Updates.IncludePrerelease` 的测试者会收到。
- 窗口位置、大小、背景透明度、文字透明度等伴随窗口本地设置由桌面程序保存。若需要重置，可删除 `%LOCALAPPDATA%\com.tyukki.mystia-steward-companion`，下次启动会重新生成。
- Windows 可能缓存旧 exe 图标。若更新后图标看起来没变，通常是系统图标缓存问题，不影响 Mod 功能。

## 快速开始

1. 启动游戏并读取存档。
2. Mod 会自动启动独立伴随窗口。若没有显示，按 `F8`。
3. 在 `概览` 页确认连接状态和推荐数据已就绪。
4. 日常查看推荐：
   - 进入 `普客` 页查看当前经营场景的普通客人推荐。
   - 进入 `稀客` 页按场景、稀客和点单 Tag 查看推荐。
   - 进入夜间经营后，在 `经营中` 页查看实时稀客/普客订单。
5. 排查问题时，再到 `设置` 页开启 `显示调试信息`，此时会出现日志页和更多诊断字段。

## 页面功能

### 概览

概览页通过页签拆分状态、库存和快捷操作：

- `状态`：显示连接状态、插件版本、运行时数据和推荐数据状态。
- `库存`：显示低库存材料和酒水，便于决定是否补货。
- `操作`：显示常用快捷键和窗口操作入口。
- 开启 `设置 -> 窗口 -> 显示调试信息` 后，会额外显示 API 状态、运行时来源和快照耗时。

### 普客

普客页按当前经营场景统计普通客人偏好，并给出料理和酒水推荐。推荐只作为经营准备参考，不会自动修改库存。

### 稀客

稀客页用于提前查看稀客推荐：

- 按经营场景筛选可选稀客。
- 选择稀客后，再选择点单料理 Tag 和酒水 Tag。
- 推荐列表会显示料理、分数、成本、厨具、基础配方、加料和酒水。
- 同一道基础料理有多个加料方案时，可在设置页控制每道基础料理最多显示几个当前排序最优的变体。
- 可在推荐行收藏料理或酒水；需要固定推荐某些未自动排到前列的料理时，使用 `自定义推荐料理` 页集中维护。

### 自定义推荐料理

自定义推荐料理用于给稀客配置固定料理方案：

- 可选择稀客、指定点单料理 Tag，或选择 `全部点单` 让该料理对该稀客所有料理 Tag 生效。
- 可选择基础料理和额外加料；游戏最多允许 5 个材料，页面会按基础配方原始材料数量和加料数量限制。
- 可启用/停用单条配置，也可开启推荐置顶并调整多条自定义配方的排序。
- 稀客页和经营中页的料理推荐区可查看当前生效的自定义配方；实际推荐仍会继续检查解锁、库存、排除材料、预算阻止和缺失厨具过滤。

### 经营中

经营中页顶部显示经营场景、推荐数据、已摆放厨具、目标厨具和通用扫描状态。下方通过 `稀客` / `普客` 页签分区。

`稀客` 页签：

- 显示当前稀客、桌位、携带金钱和点单 Tag。
- 显示当前点单推荐，严格满足点单的料理和酒水优先。
- 偏好命中但不满足点单的料理或酒水会进入同一推荐列表，并显示“偏好备选”标识。
- 可进入稀客订单专注模式，只保留当前点单推荐；专注模式支持精简显示和推荐数量设置。
- 未完成的稀客订单会通过运行时订单对象和反射回退持续校验，避免未上菜时提前从列表消失。
- 若运行时缓存偶发留下过时稀客订单，可在订单行右侧点击删除按钮清理该笔缓存。

`普客` 页签：

- 显示当前普通客人订单、料理、酒水、桌位和订单来源。
- 开启自动化后，可按独立阶段处理普客订单。
- 开启 `显示调试信息` 后，会额外显示订单 key、来源、暂停原因和内部诊断字段。

### 任务

任务页显示当前进度中的交互任务和稀客邀请：

- 任务默认显示 `可接取` 和 `可完成`，可通过多选筛选状态。
- 任务行会显示标题、角色、状态，并尽量显示角色所在场景。
- `显示额外信息` 默认关闭；开启后才显示任务 ID、来源和诊断字段。
- 稀客邀请区支持当前场景或全部场景候选，全部场景会合并当前场景已读取到的候选，也可按羁绊等级和名称搜索筛选。
- 可邀请项支持单独邀请或一次邀请全部；已邀请、羁绊不足、当前场景下当前不可见或缺少成功邀请条件的候选会保留显示但禁用按钮。
- 当前已邀请稀客会在邀请区内单独汇总显示，便于确认今晚名单。
- 邀请成功只写入游戏今晚邀请名单，不会直接刷出稀客，后续仍由游戏经营流程决定。

### 修改

修改页用于修改当前运行时库存：

- 材料和酒水列表可按名称或库存排序；单项材料或酒水支持 `-10`、`+10`、`99` 快捷修改。
- 支持 `材料全 99` 和 `酒水全 99` 批量修改当前可编辑库存。
- 修改后需要在游戏内保存，才会持久化到存档。
- 建议先备份存档；经营中修改可能与实时消耗同时发生。

### 设置

设置页通过页签管理窗口、连接、推荐、自动化和更新：

- `窗口`：背景透明度、文字透明度、焦点切换行为、切换冷却时间、始终置顶、鼠标穿透、主题、手柄导航、显示调试信息。
- `连接`：查看本机地址、LAN 状态、LAN 地址和 Token，并配置可信局域网访问。
- `推荐`：经营中订单排序、稀客专注模式默认精简、推荐权重、预算处理、排除缺失厨具、任务料理/收藏料理/收藏酒水置顶、带库存显示与排序的排除材料/酒水、同基础料理展示数量、游戏界面置顶推荐、目标厨具高亮。
- `自动化`：实验性自动化总开关、稀客/普客并发数、等待阈值、重试和回退参数。
- `更新`：检查、下载和打开安装程序。
- 开启 `显示调试信息` 后，会出现调试相关内容，并允许控制 BepInEx 原生日志窗口、日志读取和经营诊断。

### 帮助

帮助页使用目录树展示快捷键、窗口、推荐、经营中、自动化、任务、修改和日志排查说明。

### 日志

日志页仅在 `设置 -> 窗口 -> 显示调试信息` 开启后显示：

- 可读取 `BepInEx/LogOutput.log` 尾部内容。
- 可读取自动化作业日志。
- 可开启/关闭经营诊断。
- 可打开日志文件夹或导出诊断包。
- 日志读取有行数和字节上限，不会在窗口内无限累积。

## 推荐与排序规则

### 运行时数据

Mod 会尽量使用游戏当前运行时数据读取稀客、普客、料理、酒水、库存、任务和场景信息。进入游戏并读取存档后，`普客`、`稀客` 和 `修改` 页面即可使用；夜间经营准备阶段和经营中场景会继续刷新库存、解锁和订单信息。

### 厨具过滤

进入夜间经营后，Mod 会尝试读取当前场景已摆放的全部厨具。`排除缺失厨具` 开启时，推荐列表会隐藏当前场景没有对应厨具、无法制作的料理；如果未读取到厨具快照，则不会进行过滤。

### 同基础料理展示数量

`同基础料理显示` 默认只展示每道基础料理当前排序最靠前的 1 个加料变体。调高该数值后，稀客页和经营中页会保留更多同基础料理的不同加料方案；该设置只影响页面展示数量，自动化会单独选择当前可执行的目标，不会因为页面隐藏了部分变体而直接跳过可执行方案。

### 经营中订单排序

经营中订单排序默认使用 `点单顺序`。切换为 `稀客分组` 后，同一稀客的多笔订单会放在一起，稀客组之间仍按该稀客最早订单出现时间排列。

该设置会影响：

- 经营中列表。
- 稀客订单专注模式。
- 游戏界面置顶目标。
- 自动化处理的当前订单顺序。

### 推荐排序

推荐会优先满足当前点单的料理 Tag 和酒水 Tag；不满足点单但命中稀客喜好的方案会直接进入同一推荐列表，并以 `偏好备选` 标识说明不满足点单。经营中页和稀客订单专注模式会按当前显示数量截取统一列表，不再单独维护一套喜好备选区域。

满足基础条件后，推荐会按自定义推荐置顶、任务/收藏置顶和 `推荐权重` 中的启用项与权重排序。默认均衡预设会综合稀客偏好、厌恶风险、加料数量、食材压力、预计利润、酒水库存和当前厨具可做等因素。

可在 `设置 -> 推荐 -> 推荐权重` 中选择预设、启用或关闭排序项，并调整权重。排序会影响 `稀客`、`经营中`、稀客订单专注模式和自动化选单。

### 任务与收藏置顶

`设置 -> 推荐 -> 推荐设置` 中可开启 `任务料理置顶`、`收藏料理置顶` 和 `收藏酒水置顶`。自定义推荐料理的单条 `推荐置顶` 在自定义页面内配置。置顶只在解锁、库存、预算阻止、排除项和缺失厨具过滤等硬条件通过后生效；同一列表中任务料理优先于自定义置顶，自定义置顶优先于收藏料理，收藏酒水只影响酒水排序。

当前稀客存在“经营中请该角色品尝指定料理”的已接取任务时，任务料理会显示 `任务` 标识。置顶只调整推荐顺序，不会自动触发或完成任务。

### 预算和排除项

经营中稀客订单会尽量读取稀客当前剩余资金。`预算处理` 可选择阻止超预算、仅提示或忽略预算；开启后推荐行会显示预估价格和预算提示。

`排除材料` 会让料理推荐避开指定材料，基础配方和加料都会检查；`排除酒水` 会让酒水推荐跳过指定酒水。排除项下拉会显示当前库存，并可按名称或库存排序，适合临时保留材料、避免使用特定库存，或处理当前不想供应的酒水。

### 特殊经营挑战

特殊经营挑战暂不做额外规则适配。遇到怪诞料理大赛、饕餮尤魔挑战等特殊规则时，推荐和自动化仍保持标准料理、酒水链路，挑战目标与结算以游戏内提示为准。若自动化与特殊规则冲突，建议临时关闭相关自动化阶段。

## 自动化（实验性）

自动化默认关闭。需要使用时，先到 `设置 -> 自动化` 开启 `启用自动化（实验性）`，再到 `经营中` 页分别配置稀客和普客阶段开关。稀客与普客自动化互相独立。

建议先在可回退存档中测试，再长期使用。

如果同时打开 A 设备本机 Windows 伴随窗口和 B 设备 Windows / Android 伴随窗口，多个窗口可以同时查看数据和调整普通偏好；自动化会由本地 API 分配短时控制权。同一时间只有取得控制权的窗口会执行稀客或普客自动化，其他窗口会显示为仅查看，避免两边重复开锅、送达或评价。

### 自动化参数

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| 稀客并发 | `2` | 同时跟踪和处理的稀客订单数量。 |
| 普客并发 | `3` | 同时处理的普客订单数量。 |
| 最大重试 | `3` 次 | 临时失败的重试上限。 |
| 最大回退 | `2` 次 | 长时间无进展时的步骤回退上限。 |

参数会在下一轮自动化轮询生效。并发过高可能抢占厨具，最大重试和最大回退用于限制异常情况下的重复尝试。

### 稀客自动化

可配置阶段：

- `自动完成订单`：满足条件时触发订单完成评价。
- `自动送达酒水`：按推荐酒水直接送达目标订单，送达成功后扣除库存。
- `自动开始料理`：按推荐料理选择可用厨具开始制作。
- `出锅后送达`：料理完成后直接送达目标订单，送达成功后释放厨具。
- `只处理收藏料理`：自动开始料理时只使用已收藏的料理方案。
- `只处理收藏酒水`：自动送达酒水时只使用已收藏的酒水。
- `出错时暂停`：遇到非临时错误后暂停当前订单。

稀客料理和酒水会分别直接送达：酒水会从库存创建后提交到桌面，料理会在出锅后提交到桌面；只有料理和酒水都满足后才触发完整评价。

稀客订单进入自动化后，会锁定本次选择的料理、加料和酒水，避免库存变化导致后续轮询切换到另一道推荐。

自动化会锁定排序后的执行候选；如果候选是“偏好备选”料理，状态中会保留该标识。开启 `只处理收藏料理` 时只要求料理命中收藏，开启 `只处理收藏酒水` 时只要求酒水命中收藏。

### 普客自动化

需要先开启 `启用普客处理`，再配置阶段：

- `自动送达酒水`：按普客订单酒水送达并扣除库存。
- `自动开始料理`：按普客订单所需料理选择可用厨具开始制作。
- `自动送达料理`：料理完成后直接送达普客订单，送达成功后释放厨具。
- `自动完成订单`：料理和酒水都送达、订单进入待评价状态后触发游戏评价流程。
- `出错时暂停`：遇到非临时错误后只暂停对应普客订单。

普客会按首次出现顺序并发处理未评价订单。每笔订单独立记录送酒、开锅、送料理、待评价、已评价和暂停状态；同类厨具存在多个、同桌换单或同菜多单时，Mod 会尽量避免重复开锅和串单。

### QTE 奖励结算

开启任一 `自动开始料理` 后，Mod 会固定尝试完成原生 QTE 奖励结算。该流程不会打开游戏音游面板；若游戏版本变动导致入口不可用，会在自动化状态中显示诊断信息，但不会中断已开始的料理。

### 预约、等待和回退

稀客和普客会共用本轮厨具预约表。Mod 会按当前已摆放厨具数量限制同类厨具的开锅请求；同类厨具容量不足时，会进入等待态而不是立即判定失败。

经营中页会显示厨具预约，便于判断自动化是否因为厨具容量不足而等待。厨具占用、厨具冻结、订单对象暂不可读、桌面显示暂不可写等临时状态会继续等待并重试；材料不足、无可用推荐、收藏料理或收藏酒水限定不满足等情况视为硬错误。

单笔订单暂停后，可在状态卡片中点击 `重试` 继续下一轮处理；若状态明显过期，可点击 `重置` 让该订单重新判断料理、酒水和完成条件。

## 游戏界面辅助

`游戏界面置顶推荐（实验性）` 开启后，打开游戏的料理选择面板或酒水架时，Mod 会尝试把当前第一笔稀客订单的推荐料理、所需材料和推荐酒水排到列表前方。该功能只调整列表顺序，不自动点击，也不修改库存。

`目标厨具高亮（实验性）` 可独立开启。开启后，当前第一笔稀客订单的推荐料理如果有目标厨具，Mod 会尝试让经营场景中对应已摆放厨具显示黄色脉冲高亮。该功能只改变视觉提示，不会开始制作或修改库存。

## 独立伴随窗口

伴随窗口会在 Mod 加载后自动启动。它是一个独立桌面窗口，可以移动、缩放和置顶，不受游戏窗口边界限制，并会保存上次的位置和大小。

窗口关闭按钮默认隐藏到系统托盘，而不是直接退出。可以通过以下方式重新打开：

- 在游戏内按 `F8` 或 `RS Click`。
- 使用系统托盘菜单 `显示 mystia-steward-companion`。
- 再次双击 `mystia-steward-companion.exe`。

如果独立窗口当前获得焦点，再按 `F8` 或 `RS Click` 会按 `设置 -> 窗口 -> 焦点切换` 的配置切回游戏：默认会隐藏伴随窗口，也可以改为保持伴随窗口悬浮、只切回游戏焦点。保持悬浮模式更适合配合透明度常驻查看；若游戏使用独占全屏，置顶效果可能受 Windows 窗口管理限制，建议使用窗口化或无边框窗口化。

开启 `鼠标穿透锁定` 后，伴随窗口会忽略鼠标点击，便于把窗口悬浮在游戏上方查看但继续操作游戏。需要恢复窗口操作时，按 `F10` 关闭穿透，或通过托盘菜单关闭。`F8`、`RS Click`、托盘 `显示` 和托盘 `重连游戏` 都会自动关闭鼠标穿透锁定，避免窗口被找回后仍无法用鼠标操作。

### 同机连接

伴随窗口同机使用时会从游戏启动参数接收本地 API Token。若直接双击 `mystia-steward-companion.exe`，窗口会停留在未授权状态；需要连接游戏时，从游戏内按 `F8` 或托盘菜单唤起，已有窗口会接收当前游戏连接信息并自动刷新。修改 `IP:Port` 或 Token 后需要点击 `连接` 才会应用，连接失败时会逐步放慢重试，也可以点击 `停止` 暂停自动重连。

### 另一台设备连接（Windows / Android）

以下说明以 A 设备运行游戏和 Mod、B 设备只运行伴随窗口为例。两台设备需要在同一个可信局域网内，例如同一路由器下的有线或 Wi-Fi 网络。

A 设备操作：

1. 按前文安装 BepInEx #783 和 `mystia-steward-companion-bepinex.zip`。
2. 启动游戏并读取存档，等待伴随窗口自动打开；如果窗口未显示，按 `F8`。
3. 进入 `设置 -> 连接`。
4. 开启 `允许局域网设备连接`。开关会立即应用；只有修改 `LAN 监听地址` 时才需要点击 `应用`。
5. 确认 `LAN 状态` 显示已开启，并复制 `LAN 地址` 和 Token。
6. 如果 `LAN 地址` 未生成，先确认 A 设备有可用的局域网 IPv4，例如 `192.168.x.x`、`10.x.x.x` 或 `172.16.x.x` 到 `172.31.x.x`。多网卡、虚拟网卡或 VPN 可能让自动检测选到错误地址，此时可以在 `LAN 监听地址` 中填写 A 设备实际的局域网 IPv4。
7. 首次开启或更换端口后，如果 B 设备连接超时，在 Windows 防火墙中允许 A 设备上的 `32145` 端口或游戏/伴随窗口相关程序入站。

B 设备操作：

1. Windows 设备从 Release 下载 `mystia-steward-companion-companion-windows-x64.exe`，下载后直接双击运行；如果当前版本还没有独立 EXE，也可以从完整 Mod 包的 `mystia-steward-companion/companion/` 目录取出 `mystia-steward-companion.exe`。
2. Android 设备从 Release 下载对应 CPU 架构的 APK 并安装。普通手机和平板优先选择 `mystia-steward-companion-android-arm64-v8a.apk`，旧设备再选择 `mystia-steward-companion-android-armeabi-v7a.apk`。若系统提示未知来源，需要只对本次下载来源临时允许安装。
3. 启动 B 设备上的伴随窗口。
4. 在窗口顶部连接区填写 A 设备复制出的 `LAN 地址`，例如 `http://192.168.1.20:32145`。
5. 填写 A 设备复制出的 Token。
6. 点击 `连接`，等待顶部连接状态变为已连接。

连接限制和排查：

- A 设备本机地址 `http://127.0.0.1:32145` 始终保留。即使 LAN 配置错误，仍可在 A 设备本机回到 `设置 -> 连接` 修改。
- B 设备不能使用 `127.0.0.1` 连接 A 设备，因为它只代表 B 设备自己。
- 不要填写 `0.0.0.0`，也不要通过公网端口映射暴露本地 API。
- 除 `/health` 外，局域网访问都需要 Token。Token 错误通常表现为未授权或连接后无法读取数据。
- 连接配置和 Token 重置只能在 A 设备本机伴随窗口操作，B 设备不能远程修改这些安全配置。
- `F8`、`RS Click` 和自动启动只控制 A 设备本机窗口，不能远程唤起 B 设备窗口。
- 多个伴随窗口可以同时连接查看；自动化同一时间只允许一个窗口取得控制权，未取得控制权的窗口不会执行自动化动作。
- Android 版没有桌面托盘、置顶、鼠标穿透、窗口焦点切换和游戏关闭自动退出能力；这些能力只适用于 A 设备本机的 Windows 伴随窗口。
- Android 版支持竖屏和横屏；手机竖屏下顶部页签栏可横向滑动，焦点或当前页签会自动滚入可视区域，平板和横屏会尽量接近桌面布局；内容会避开系统状态栏和导航栏安全区域。

A 设备本机 Windows 伴随窗口在游戏关闭或 Mod 卸载时会自动退出；B 设备独立窗口和 Android 版会保留界面并进入重连状态。断线后窗口只保留最近一次只读数据用于查看，自动重连期间优先做轻量健康检查，不再执行自动化、置顶目标发布等会写入游戏运行时的后台动作。

## 配置与数据位置

主配置文件：

```text
BepInEx/config/com.tyukki.mystia-steward-companion.cfg
```

常用用户数据：

```text
BepInEx/config/MystiaStewardCompanion/favorites.json
BepInEx/config/MystiaStewardCompanion/automation-jobs.log
BepInEx/config/mystia-steward-companion/night-business-diagnostics.log
```

常用配置项：

| 配置项 | 默认值 | 说明 |
| --- | --- | --- |
| `ToggleKey` | `F8` | 游戏与独立窗口焦点切换热键。 |
| `ControllerToggleKey` | `JoystickButton9` | 手柄焦点切换热键，常见映射为 `RS Click`。 |
| `Companion.AutoLaunch` | `true` | 插件加载后自动启动伴随窗口。 |
| `Companion.ExecutablePath` | 空 | 伴随窗口可执行文件路径；留空时自动查找。 |
| `LocalApi.AllowLanConnections` | `false` | 是否允许可信局域网设备连接；本机回环连接始终启用。 |
| `LocalApi.LanHost` | `auto` | LAN 监听地址；`auto` 会监听检测到的私网 IPv4，也可指定本机私网 IPv4。 |
| `LocalApi.Port` | `32145` | 本地 API 端口。 |
| `LocalApi.ExposeLogs` | `true` | 允许伴随窗口读取 `BepInEx/LogOutput.log`。 |
| `LocalApi.MaxLogLines` | `300` | 日志页每次最多返回的日志行数。 |
| `LocalApi.MaxLogBytes` | `262144` | 日志页每次最多扫描的日志尾部字节数。 |
| `BepInEx.DisableConsoleLogWindow` | `true` | 将 BepInEx 控制台日志窗口设为下次启动关闭。 |
| `BepInEx.HideConsoleWindow` | `true` | Mod 加载后尝试隐藏当前 Windows 控制台窗口。 |
| `Diagnostics.EnableNightBusinessDiagnostics` | `false` | 是否写入夜间经营诊断日志。 |
| `SetConsoleUtf8` | `true` | Mod 加载后尝试把 Windows 控制台切换到 UTF-8。 |

如果从旧版本升级，Mod 会在新配置不存在时自动复制旧配置到当前文件名。

## 日志与诊断

普通使用不需要开启诊断。遇到识别、推荐或自动化问题时，建议按以下顺序收集信息：

1. 打开 `设置 -> 窗口 -> 显示调试信息`。
2. 进入 `日志` 页，开启日志读取和经营诊断。
3. 复现问题。
4. 导出诊断包，或提供 `BepInEx/LogOutput.log`、经营诊断日志和伴随窗口日志页内容。

开启经营诊断后，除 `night-business-diagnostics.log` 外，还会按需写入运行时固定数据快照：

- `runtime-static-data.log`：稀客映射和运行时别名归一化来源。
- `runtime-tags.log`：料理/酒水标签文本和 TagRule。
- `runtime-database-diff.log`：食材、酒水、料理运行时数据对照。
- `runtime-guests.log`：普客、稀客和映射稀客。
- `runtime-izakayas.log`：经营场景、场景标签和客人池。

这些文件不是游戏启动瞬间立即生成，而是在诊断开启且对应运行时数据扫描触发时写入。日志顶部 `Complete: True` 且 `Status` 中各项计数大于 0，通常表示本次固定数据读取成功。

自动化诊断会写入 `BepInEx/config/MystiaStewardCompanion/automation-jobs.log`，文件超过约 1 MB 时自动轮换为 `.1`。排查“已开始制作但未直接送达”“订单长时间等待”“重复开锅”等问题时，优先提供诊断包或该日志。

## 故障排查

### `F8` 无法打开独立窗口

确认 `mystia-steward-companion.exe` 位于：

```text
BepInEx/plugins/mystia-steward-companion/companion/
```

如果放在其他位置，请在 `Companion.ExecutablePath` 中填写绝对路径。若窗口已打开，`F8` 会在游戏和独立窗口之间切换。

### 伴随窗口鼠标点不到

通常是开启了鼠标穿透锁定。按 `F10` 关闭，或通过托盘菜单选择 `关闭鼠标穿透`。

### 一直显示运行时数据不可用

先确认已经进入游戏并读取存档。若仍不可用，开启 `显示调试信息`，进入 `日志` 页查看连接状态和 `LogOutput.log`。

### 经营中没有稀客或点单

确认游戏内稀客已经入座并完成点单。若仍不显示，开启 `显示调试信息` 和经营诊断，查看经营中页扫描状态与日志页内容。

### 自动化卡住或重复等待

先查看经营中页对应订单的自动化状态和厨具预约。常见原因包括厨具占用或冻结、材料不足、订单或桌面对象暂不可读、推荐为空、收藏料理或收藏酒水限定不满足。必要时对单笔订单点击 `重试` 或 `重置`。

### 夜间经营订单较多时卡顿

先开启 `显示调试信息`，在概览页或经营中页查看快照耗时。若卡顿同时伴随多笔订单自动化，建议临时降低稀客并发或普客并发，或关闭暂时不需要的自动化阶段。仍无法定位时，到日志页导出诊断包，并记录当时稀客/普客订单数量、开启的自动化阶段和是否开启经营诊断。

### 启动时仍短暂出现控制台

Mod 加载时控制台已经由 BepInEx 创建，本次启动只能尝试隐藏；`BepInEx.DisableConsoleLogWindow=true` 写入的配置会在下一次启动生效。需要临时查看原生控制台时，开启 `显示调试信息` 后到设置页切换。

### 控制台早期中文乱码

Mod 只能在自身加载后切换 UTF-8，不能修复 BepInEx preloader 已经输出的日志。日常建议关闭控制台；排查原生控制台问题时再临时开启。

### 想重置 Mod 设置

关闭游戏和伴随窗口后，按需要删除以下内容：

- 重置插件配置：`BepInEx/config/com.tyukki.mystia-steward-companion.cfg`
- 重置收藏和自动化日志：`BepInEx/config/MystiaStewardCompanion/`
- 重置伴随窗口位置、大小和本地偏好：`%LOCALAPPDATA%\com.tyukki.mystia-steward-companion`

删除后下次启动会重新生成默认配置。
